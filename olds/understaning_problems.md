# KPMG AI Builder 과제 진행 계획

## 1. 과제 목표 이해

- KPMG AI Builder 후보자를 잘 평가할 수 있는 실용적인 산출물을 만든다.
- 정답이 정해진 과제가 아니므로, 내가 생각하는 "AI Builder를 잘 평가한다"의 기준을 먼저 정의한다.
- 핵심 평가는 채용 도구 자체가 아니라, 모호한 문제를 구체적인 방향으로 좁히고 실제로 무언가를 만드는 능력이다.
- 전체 작업 시간은 빌드, 설명 준비, 영상 녹화까지 포함해 최대 3시간이다.

## 2. 내가 만들어야 할 산출물

필수 제출물은 아래 2가지다.

- Artifact: 작동하는 프로토타입, 워크플로우, 평가 프레임워크, 도구, 시뮬레이션, 리뷰 프로세스 중 하나
- 3분 영상 링크: 첫 1분은 만든 것 시연, 다음 2분은 접근 방식과 판단 설명

추가 제출 형식:

- PDF 파일 1개로 제출
- 파일명: `FirstName_LastName_AIBuilder`
- PDF 안에 3분 영상 링크 포함
- PDF 안에 데모 또는 artifact 링크 포함

## 3. 추천 방향

이번 과제에서는 짧은 시간 안에 보여주기 좋도록 아래 방향이 적합하다.

**AI Builder Candidate Evaluation Workbench**

- 후보자의 과제 제출물이나 인터뷰 답변을 평가하는 간단한 웹 기반 평가 도구
- AI Builder 역할에 맞는 평가 기준을 직접 정의
- 평가자는 후보자의 답변을 입력하고, 루브릭별 점수와 코멘트를 남길 수 있음
- 최종적으로 강점, 리스크, 후속 질문, 추천 여부를 요약

이 방향을 선택하는 이유:

- KPMG가 원하는 "모호한 문제를 구조화하는 능력"을 직접 보여줄 수 있음
- 엔터프라이즈 환경에서 중요한 공정성, 거버넌스, 투명성, 후보자 경험을 반영하기 좋음
- 3시간 안에 HTML/CSS/JS 또는 간단한 웹앱으로 만들 수 있음
- 영상 1분 시연이 쉬움

## 4. 평가 기준 정의

AI Builder 후보자를 평가할 때 아래 기준을 사용할 수 있다.

- Ambiguity Framing: 모호한 문제를 어떻게 정의하고 범위를 좁혔는가
- Practical Building: 실제로 작동하거나 검토 가능한 산출물을 만들었는가
- AI Leverage: AI 도구를 목적에 맞게 사용했고, 본인이 이해하고 설명할 수 있는가
- Enterprise Judgment: 보안, 리스크, 거버넌스, 확장성, 운영 현실을 고려했는가
- User and Stakeholder Thinking: 채용팀, 후보자, 비즈니스 리더 등 이해관계자의 요구를 균형 있게 고려했는가
- Communication: 접근 방식, 트레이드오프, 한계, 다음 단계를 명확히 설명했는가
- Responsible AI: 공정성, 편향, 투명성, 개인정보, 후보자 경험을 고려했는가

## 5. 빌드할 기능 체크리스트

최소 기능:

- 후보자 이름 또는 제출물 제목 입력
- 후보자 산출물 요약 입력
- 평가 기준별 점수 선택
- 기준별 코멘트 입력
- 자동으로 총점 또는 평균 점수 계산
- 강점, 리스크, 후속 질문 작성 영역
- 최종 추천 상태 표시: Strong Yes / Yes / Maybe / No
- 샘플 후보자 데이터 1개 제공

있으면 좋은 기능:

- 루브릭 설명을 각 항목 옆에 표시
- 점수에 따라 시각적 색상 표시
- 평가 결과를 복사할 수 있는 요약문 생성
- 공정성 체크 항목 제공
- "AI 사용 방식"을 기록하는 섹션 제공

시간이 부족하면 제외할 기능:

- 로그인
- 데이터베이스 저장
- 실제 AI API 연동
- 복잡한 관리자 화면
- 여러 후보자 관리 기능

## 6. 3시간 작업 순서

### 0:00-0:20 문제 정의

- AI Builder를 잘 평가한다는 것이 무엇인지 5-7개 기준으로 정의한다.
- 평가 대상은 실제 개인정보가 아닌 synthetic/sample 후보자 데이터로 제한한다.
- 내가 만들 artifact의 이름과 사용 시나리오를 정한다.

### 0:20-1:40 프로토타입 제작

- 단일 페이지 웹앱 또는 문서형 워크벤치 제작
- 루브릭, 점수 입력, 코멘트 입력, 결과 요약 기능 구현
- 샘플 데이터 추가
- UI는 과하게 꾸미기보다 검토자가 바로 이해할 수 있게 구성

### 1:40-2:10 README 또는 설명 문서 작성

- 무엇을 만들었는지
- 어떻게 사용하면 되는지
- 어떤 기준으로 후보자를 평가하는지
- 의도적으로 제외한 것과 이유
- 리스크와 개선 방향

### 2:10-2:35 영상 스크립트 준비

- 첫 1분: artifact 시연
- 다음 2분: 문제 정의, 접근 방식, 트레이드오프, 리스크, AI 사용 방식 설명

### 2:35-3:00 영상 녹화 및 PDF 제출 파일 준비

- 3분 영상 녹화
- 영상 링크 생성
- 데모 또는 파일 링크 준비
- PDF에 링크와 간단한 설명 포함
- 파일명을 `FirstName_LastName_AIBuilder.pdf` 형식으로 저장

## 7. 영상 구성안

### 첫 1분: 시연

- "제가 만든 것은 AI Builder Candidate Evaluation Workbench입니다."
- 샘플 후보자 선택 또는 입력
- 루브릭별 점수와 코멘트 입력 화면 보여주기
- 총점, 추천 상태, 후속 질문이 생성되는 흐름 보여주기
- 공정성 또는 거버넌스 체크 섹션 보여주기

### 다음 2분: 설명

- 문제를 "최고의 정답 찾기"가 아니라 "다양한 배경의 후보자를 공정하고 실용적으로 평가하는 구조 만들기"로 정의했다고 설명
- 평가 기준은 AI Builder 역할의 핵심인 모호성 처리, 실제 빌드 능력, AI 활용, 엔터프라이즈 판단, 커뮤니케이션으로 구성했다고 설명
- 트레이드오프: 짧은 시간 때문에 실제 AI API, 로그인, 데이터 저장은 제외하고 평가 흐름과 판단 구조에 집중했다고 설명
- 리스크: 루브릭도 편향될 수 있으므로 여러 평가자 캘리브레이션, 점수 근거 기록, 후보자 경험 검토가 필요하다고 설명
- AI 사용: 아이디어 구조화, 루브릭 초안, UI/문서 작성 보조에 AI를 사용했지만 최종 기준과 범위 결정은 내가 했다고 설명
- 개선 방향: 실제 제출물 업로드, 다중 평가자 비교, 편향 점검 리포트, 인터뷰 질문 추천 기능 추가

## 8. 제출 전 확인사항

- [ ] Artifact가 열리고 검토 가능한가
- [ ] 샘플 데이터만 사용했는가
- [ ] 개인정보나 confidential data가 없는가
- [ ] README 또는 사용 설명이 있는가
- [ ] 3분 영상 링크가 작동하는가
- [ ] 데모 또는 artifact 링크가 작동하는가
- [ ] PDF 안에 영상 링크와 artifact 링크를 모두 넣었는가
- [ ] PDF 파일명이 요구 형식에 맞는가
- [ ] 3시간 안에 끝낼 수 있도록 범위를 유지했는가

## 9. 최종 제출물에 들어갈 짧은 설명 예시

I built an AI Builder Candidate Evaluation Workbench: a lightweight rubric-based review tool that helps hiring teams evaluate candidates across ambiguity framing, practical building, AI leverage, enterprise judgment, stakeholder thinking, communication, and responsible AI. I focused on making the evaluation transparent, inspectable, and usable within a constrained hiring workflow rather than building a fully automated scoring system.

## 10. 내가 특히 강조할 메시지

- 이 과제의 핵심은 채용 자동화가 아니라 판단 구조를 설계하는 것이다.
- AI Builder는 정해진 문제를 푸는 사람보다, 모호한 맥락에서 무엇을 만들 가치가 있는지 정의하고 빠르게 검증하는 사람이다.
- 따라서 artifact도 후보자를 단순 점수화하기보다, 평가자가 근거 있는 판단을 하도록 돕는 도구로 설계한다.
- KPMG 같은 기업 환경에서는 속도와 혁신뿐 아니라 리스크, 보안, 거버넌스, 공정성, 후보자 경험을 함께 고려해야 한다.
